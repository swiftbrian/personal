The Agentic
Platform Model

A shared mental model for how SafetyCulture's platform evolves — from a tool that humans use to gather data, to an intelligence layer that agents use to guide humans.

01
The Fundamental Shift

The agentic era doesn't just add new features to our existing model — it inverts the relationship between humans and software. Understanding this inversion is the starting point for everything else in this document.

Today — Human-driven
Humans use software
to collect data
👤
Workers initiate every interaction. An inspection happens because a person opens the app, selects a template, and fills it in. The platform is passive — it waits.
📋
Templates are the product. The 10,000-template library is the primary value — structured forms that capture what humans observe.
📊
Data flows up. Information moves from worker → form → database → report → manager. It's slow, and a lot is lost in translation.
⚡
Insights are retrospective. By the time a manager sees a pattern, the window to prevent an incident has usually passed.
→
Tomorrow — Agent-driven
Agents use our data
to guide humans
🤖
Agents initiate most interactions. The platform continuously monitors, detects, and acts. Humans confirm, decide, and resolve — not collect and report.
🧠
Operational intelligence is the product. The value is what we know — 10 years of physical-world data that no competitor can replicate — not the templates that capture it.
🔄
Intelligence flows both directions. Agents surface the right information to the right person at the right moment. Context moves to the worker, not just the manager.
🎯
Insights are predictive. The Five Star Rating drops before an incident occurs. The agent intervenes. The cycle of reactive response is broken.

03
How Data Flows

A single real-world event — a temperature excursion in a cold store at 2am — travels through every layer of the platform. This trace shows exactly what happens, which layer handles it, and how long it takes.

Scenario
Cold store temperature breach — 2:14am, Melbourne Site B
Data Platform
↓
IoT temperature sensor records 6.2°C — above the 4°C threshold
Sensor reading is timestamped and written to the data platform. This is the only human-free trigger in the old model — and historically, it would sit in a log until someone checked it in the morning.
t = 0s
Knowledge Graph
↓
Event is enriched with operational context
The Knowledge Graph attaches context: this is Cold Store 2, last inspected 18 hours ago, used for dairy products with a regulatory limit of 4°C. Site manager is Jamie M. The on-call inspector is Tom Chen. Previous breach at this asset: 4 months ago.
t < 2s
Intelligence
↓
Risk models classify severity and update the Five Star Rating
The anomaly detection model flags this as a high-risk event. The Five Star Rating engine recalculates Site B's score — it drops from 4.1 to 3.8. The predictive model checks: is this a one-off or a trend? Previous breach + maintenance record suggests equipment fault, not procedural failure.
t < 5s
Agent Layer
↓
Site Safety Agent reasons, plans, and acts
The agent determines: this requires immediate inspection at shift start. It pre-fills an inspection template for Cold Store 2 (seal check, compressor status, temperature log review). It drafts a shift brief for Jamie M. It checks if the breach duration requires regulatory notification — it does not yet, but will if unresolved by 6am. It queues an escalation trigger.
t < 8s
Surfaces
↓
Jamie opens Co-Pilot at 7:02am — brief is already waiting
"Your site's Five Star Rating dropped to 3.8 overnight. Cold Store 2 had a temperature excursion at 2:14am — inspection pre-filled and ready." Jamie approves with one tap. Tom Chen receives the assignment. By 7:48am, the seal is replaced, temperature is stable, score recovers to 4.1.
Resolution: 94 min
Feedback Loop
↺
Resolution data flows back — the system learns
The corrective action (seal replacement) is written back to the Knowledge Graph. The asset's maintenance history is updated. The Intelligence layer notes: seal failure follows temperature excursion at this asset type after ~18 months. It flags other assets of the same type for proactive inspection. The next breach is prevented before it happens.
Continuous learning
04
The Agent Layer, Unpacked

Agents are not chatbots or search tools. They are autonomous reasoning loops — they observe, plan, act, and verify, without waiting for a human to initiate. Each agent has a defined scope, a set of triggers, and a set of permissible actions.

🏗️
Core Agent
Site Safety Agent
Monitors a single site continuously. Owns the Five Star Rating for that site and takes action when it drops. The primary interface between the platform and the frontline.
Rating drops below threshold → brief site manager
Inspection overdue → assign nearest qualified worker
Shift start → generate personalised brief
Incident filed → begin corrective action loop
📋
Core Agent
Compliance Agent
Maintains continuous audit-readiness. Monitors documentation gaps, overdue corrective actions, and regulatory deadlines. Assembles evidence packs on demand.
Corrective action overdue → escalate with 48hr notice
Audit announced → assemble evidence pack
Regulatory change → flag affected workflows
Documentation gap → notify responsible owner
🔄
Core Agent
Corrective Loop Agent
Closes the gap between issue detection and resolution. Owns the corrective action lifecycle — from creation through assignment, verification, and sign-off.
Issue detected → create action, assign owner
Action unstarted after 24hrs → send reminder
Action completed → trigger verification inspection
Verification passed → close and update rating
📈
Core Agent
Risk Prediction Agent
Surfaces risk before it becomes an incident. Analyses trends across inspections, assets, and team behaviour to generate proactive alerts and recommended interventions.
Site trending downward → alert portfolio manager
Asset approaching failure pattern → schedule maintenance
Near-miss spike → identify root cause
Weekly → generate portfolio risk brief
🤝
Core Agent
Handover Agent
Eliminates shift handover as a failure point. Automatically assembles end-of-shift summaries, outstanding items, and next-shift priorities from the day's activity.
Shift ending → compile activity summary
Open items → prioritise and assign to next shift
Handover approved → notify incoming shift lead
Critical item unresolved → escalate immediately
🔧
Partner / Custom
Custom Agents (SDK)
Built by customers, FDEs, or partners using the Agent SDK. Compose with SafetyCulture's data layer. Examples: Confined Space Entry Agent, Supplier Audit Agent, Brand Standards Agent.
Customer-defined triggers and actions
Access to full Knowledge Graph context
Can invoke core agents as sub-agents
Revenue-sharing model via Marketplace
How agents orchestrate each other
Agents don't operate in isolation. They form a multi-agent system — the Site Safety Agent can invoke the Corrective Loop Agent, which can invoke the Compliance Agent to check regulatory requirements, which feeds back to the Risk Prediction Agent. The platform becomes a nervous system, not just a sensor.

Each agent operates within a defined trust boundary: they can read any data they're permitted to access, propose any action within their scope, and execute only after human approval thresholds are met. The human is always in the loop — but for exceptions, not every step.
Example: End-to-end corrective loop
1
Site Safety Agent detects Five Star Rating drop. Identifies fire door inspection as the cause.
2
Compliance Agent checks: is this a regulatory breach? Confirms 48hr notification window applies.
3
Corrective Loop Agent creates action, identifies nearest certified inspector, assigns with deadline.
4
Frontline Co-Pilot notifies the assigned inspector with context, pre-filled checklist, and permit.
5
Corrective Loop Agent verifies completion. Compliance Agent confirms documentation. Rating updates.
6
Risk Prediction Agent logs pattern. Flags similar assets for proactive review. Closes the loop.

Personalisation & Context

A generic platform is a commodity. The intelligence layer only becomes irreplaceable when it knows who you are, what industry you're in, and what your organisation specifically needs. Personalisation is not a feature — it's a cross-cutting architectural concern that shapes every agent, every surface, and every rating calculation.

1
Level 1 · SafetyCulture configures
Industry Vertical
The platform arrives knowing your industry. Pre-loaded standards, regulations, agent sets, and Five Star Rating weightings — all specific to your sector. You don't configure what matters. We already know.
Construction: machinery guarding, permit-to-work, scaffold inspection weighted highest
Food manufacturing: HACCP, temperature control, allergen management agents active by default
Hospitality: fire safety, food hygiene, public liability drives the rating
Mining: fatigue management, blast safety, isolation procedures pre-loaded
2
Level 2 · Customer configures
Company Configuration
Your specific SOPs, org structure, risk thresholds, escalation paths, and vocabulary. The agent speaks your language — it says "notify your Area Manager" not "contact a supervisor", because it knows your org chart.
Custom escalation chains mapped to org hierarchy
Proprietary SOP library ingested and indexed in Knowledge Graph
Company-specific risk thresholds override vertical defaults
Brand voice and terminology reflected in agent communication
3
Level 3 · Platform learns
Individual Context
Role, certifications, shift history, site assignment, past behaviour. The Co-Pilot brief for a first-week worker looks completely different from the one for a five-year site manager. Context is earned, not configured.
Certified vs uncertified workers receive different procedure guidance
Shift history informs handover detail and urgency
Past behaviour patterns surface relevant proactive alerts
Language preference, role scope, site familiarity all shape the brief

07
Architecture Principles

Six principles that constrain every design decision in the agentic platform. These are the tests every team should apply before building.

01
Agents propose. Humans decide.
Agents can observe, reason, draft, assign, and escalate. They cannot permanently delete, submit to regulators, or act outside defined trust boundaries without human approval. The loop always closes with a person.
02
Data is the moat. Protect it.
Every architectural decision is evaluated against one question: does this compound our data advantage, or does it commoditise it? We share intelligence, not raw data. We open APIs, not databases.
03
New above, stable below.
The new agent platform is built above the existing system, not through it. Existing customers are never the guinea pigs for architectural change. Stability is a feature, not a constraint.
04
Context makes agents useful.
A generic AI agent is a commodity. An agent that knows this worker, this site, this asset's history, and the regulations for this jurisdiction is irreplaceable. The Knowledge Graph is what makes our agents worth building.
05
The rating is the interface.
The Five Star Rating is not just a metric — it's the primary trigger, the primary feedback loop, and the primary interface between the agent layer and both workers and managers. Every agent read from it and writes to it.
06
Build for the ecosystem.
Every component of the agent platform — the SDK, the Knowledge Graph API, the Marketplace — is designed to be used by others, not just us. Platform economics require that others can build on top of what we build.
The architecture described here is not a roadmap for adding AI features. It is a blueprint for becoming a fundamentally different kind of company — one whose primary product is operational intelligence, and whose primary interface is an agent that knows more about your physical world than any human team ever could.
